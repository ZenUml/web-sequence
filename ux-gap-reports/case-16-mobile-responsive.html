Identified Gaps
GAP-16-001 High

No responsive layout — desktop split-pane renders at full width on 390px mobile

The entire application renders as a desktop layout with no breakpoint adaptations for mobile screen widths. The split-pane editor+canvas design assumes a wide viewport and fails catastrophically on narrow screens:

  • CSS audit reveals only 3 meaningful mobile rules under max-width: 600px/768px: body { font-size: 70% }, .main-header { overflow-x: auto }, and .hide-on-mobile { display: none }
  • The new Tailwind-based UI (sidebar, editor pane, canvas pane) has zero mobile-specific CSS — it was built desktop-first with no mobile adaptation
  • No hamburger menu, no bottom navigation sheet, no drawer/off-canvas pattern
  • No viewport-aware layout switching (e.g., editor-only mode on narrow screens)
  • The width=device-width, initial-scale=1 viewport meta tag signals mobile intent — the CSS doesn't follow through
Mobile-first design Responsive design fundamentals WCAG 1.4.10: Reflow (320px width requirement)
GAP-16-002 High

Diagram canvas reduced to 55px sliver — completely unusable on mobile

At 390px width, the layout allocates: ~50px sidebar + ~200px editor pane + ~55px canvas sliver + ~85px overflow. The canvas is visible as a 55px strip showing only the leftmost edge of the first participant box:

  • Only the letter "A" (first participant) is partially visible — the full diagram is cut off
  • No horizontal scroll on the canvas container — the rest of the diagram is unreachable
  • Canvas footer controls (zoom in/out, info button) are visible but the zoom percentage and "ZenUML.com" watermark are clipped
  • No touch-to-pan or pinch-to-zoom gesture on the canvas for mobile users
  • The core product value (viewing sequence diagrams) is completely destroyed on mobile

GAP-16-003 High

All export buttons (Present, PNG, Copy PNG) disappear completely on mobile

The bottom action bar contains "Present | ↓ PNG | ⎘ Copy PNG" buttons on the right side of the screen. At 390px, these buttons overflow beyond the viewport and are completely inaccessible:

  • The bottom bar has no horizontal scroll
  • Only the Settings gear (far left) and "Page 1" tab (partially) are visible
  • No touch-swipe to reveal hidden buttons
  • No overflow indicator (no "..." or arrow suggesting more content)
  • The "+ Add Page" button is also cut off — users cannot create new pages on mobile
  • Even if export weren't login-gated (see Case 15), mobile users couldn't access it anyway
WCAG 1.4.10: Reflow Nielsen #3: User Control & Freedom Mobile layout overflow
GAP-16-004 Medium

Diagram title wraps to 3 lines in header — wastes vertical space and looks broken

The header title element contains "My Auth Flow Diagram" (from the previous session). At 390px, this wraps into three lines: "My Auth / Flow / Diagram". The header height inflates significantly:

  • A centered header title that wraps is unexpected — most apps truncate with ellipsis
  • No text-overflow: ellipsis or max-width constraint on the title element
  • The inflated header further reduces the already-limited vertical space for the editor
  • Short diagram names (e.g., "Diagram") wouldn't wrap, making behavior unpredictable and name-length-dependent
Nielsen #4: Consistency & Standards Text overflow handling Vertical space budget
GAP-16-005 Medium

Insert toolbar clips to 4 icons — 50%+ of editing shortcuts unreachable on mobile

The insert toolbar above the editor shows 8+ icon buttons (New participant, Async, Sync, Return value, Self message, New instance, Conditional, Loop, and more). At 390px, only the first 4 fit:

  • No overflow scrolling on the toolbar row
  • No "more" button or expandable overflow menu
  • Hidden icons include Conditional (Alt/Else), Loop, Group — complex constructs that benefit most from toolbar help
  • The toolbar has no overflow-x: auto — clipped content is simply invisible

GAP-16-006 Low

No touch-optimized interactions on diagram canvas — pinch-zoom and swipe-pan absent

Even at desktop sizes, the diagram canvas lacks scroll-to-zoom and drag-to-pan (documented in Case 09). On mobile these gaps are worse because touch users have no mouse wheel alternative:

  • No pinch-to-zoom gesture on the canvas
  • No swipe-to-pan to navigate large diagrams
  • The only zoom controls are two small ⊕/⊖ buttons in the canvas footer — too small for touch (sub-24px targets)
  • Touch users cannot explore diagrams that extend beyond the visible canvas
