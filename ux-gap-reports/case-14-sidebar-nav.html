Identified Gaps
GAP-14-001 High

All 6 sidebar icons missing aria-label — screen reader announces ligature text

Every sidebar button relies solely on the HTML title attribute for labeling. The button text content is the raw Material Icons ligature string (folder_open, code_blocks, quick_reference, etc.). A screen reader announces these strings verbatim, providing zero semantic value.

  • title attributes are not announced by most screen readers when focus is managed
  • title tooltips are invisible on touch/mobile devices
  • Screen reader output: "folder underscore open, button" instead of "My Library, button"
  • WCAG 4.1.2 Name, Role, Value — Level A failure for all 6 interactive elements

GAP-14-002 High

Library panel cannot be dismissed — Code Editor icon inert when Library is open

Once "My Library" is opened, the user has no working path back to the Code Editor:

  • Clicking the "My Library" folder icon again does not toggle it closed
  • Clicking the "Code Editor" icon while the Library panel is open does nothing — the panel state doesn't change
  • The "Collapse panel" chevron button (‹) visually disappears from the DOM after click but the panel content remains fully visible
  • Escape key closes modals but has no effect on the Library panel
  • No close (×) button on the Library panel itself
  • The only reliable recovery is refreshing the page

GAP-14-003 Medium

540px dead zone splits sidebar into two visually disconnected icon clusters

The 6 sidebar icons are distributed in two polarised groups with no visual grouping, separator, or label to explain the split:

  • Top cluster: My Library (y=61px) + Code Editor (y=119px) — workspace actions
  • Bottom cluster: Keyboard Shortcuts (y=661px), Cheatsheet (y=719px), Language Guide (y=776px), Settings (y=834px) — help/config
  • ~540px gap between clusters with nothing visible — looks broken, not intentional
  • On shorter viewports (<835px tall), Settings (y=834px) is clipped off-screen with no scroll affordance
  • Violates Gestalt Proximity — icons that belong together should sit together; unrelated icons should be separated by a visible divider, not empty space

GAP-14-004 Medium

Same icon strip, three different interaction patterns with no visual differentiation

The 6 icons look identical in style but trigger fundamentally different interaction types, with no affordance distinguishing them:

  • Panel swap: My Library replaces the editor in-place
  • Modal: Keyboard Shortcuts and Cheatsheet open overlay dialogs
  • External link: Language Guide navigates away to zenuml.com/docs in a new tab — no external link icon (↗), no tooltip warning, no visual distinction

Users cannot predict the outcome of clicking any icon. An external link inside an icon-only sidebar is a particularly dangerous pattern — the user loses context with no warning.
