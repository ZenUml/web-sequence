🔴 Identified UX Gaps

GAP-01-001
HIGH

No onboarding or empty-state guidance

New users land on a mostly-black screen with a single letter "A" in the editor and a tiny diagram. There is no welcome message, tutorial prompt, or "get started" example.

Nielsen #6 — Recognition over recall
Show a rich default example (3-4 participants, arrows, return values) with an inline comment block explaining the syntax.
GAP-01-002
HIGH

Sidebar icons have no tooltips or labels

The 6 sidebar icons (folder, code editor, keyboard, document, library, settings) have no hover tooltip and no visible label. Users must click each one to discover what it does.

Nielsen #6 — Recognition over recall
Add tooltip on hover (100ms delay) and optionally a collapsible label rail. Modern tools (Figma, Linear, Notion) all label their nav icons.
GAP-01-003
HIGH

Preview canvas wastes ~75% of available width

The diagram renders in a small floating frame (≈260px wide) pinned to the top-left of a large dark canvas. The remaining right-side space is completely unused.

Fitts's Law — targets should be large and proximate
Make the preview fill the available pane width and auto-fit the diagram to the viewport (like Mermaid Live Editor, draw.io). Add a "fit to screen" button.
GAP-01-004
MEDIUM

Settings modal mislabeled as "Editor"

The gear icon in the sidebar opens a dialog titled "Editor", not "Settings". Users looking for app settings will be confused; "Editor" implies code editor preferences only.

Nielsen #4 — Consistency & standards
Rename the modal to "Settings" or split into "Editor Preferences" and "App Settings". The icon should match the modal title.
GAP-01-005
MEDIUM

Editor toolbar icons are unlabeled

8+ icon buttons appear above the code editor (participant, arrow types, loops, alt/else blocks) with no labels, no hover tooltips, and no visual grouping. The syntax they insert is non-obvious from the icon alone.

Nielsen #6 — Recognition over recall
Add tooltips showing the inserted syntax snippet. Group related icons with a divider. Consider a label on hover like "Insert if/else block (Alt)".
GAP-01-006
MEDIUM

"Preserve console logs" exposed in user settings

A debug/developer option sits alongside user-facing preferences (Theme, Font, Line wrap). This is noise for the vast majority of users and reduces trust.

Progressive disclosure — hide complexity from non-technical users
Move debug options behind an "Advanced" expandable section or remove from the UI entirely unless explicitly requested.
GAP-01-007
LOW

Title edit affordance is easy to miss

The diagram title "Data Processing Flow" has a small pencil icon to the right that only appears at a glance. There's no hover state or underline hint that the title is editable in-place.

Nielsen #1 — Visibility of system status
Show a hover underline on the title text itself (not just the icon) so users know it's clickable. Industry standard: clicking the title text should open the inline editor.
